Innovative Networking Solutions for Small Offices: A Showcase of My Expertise and Projects

Network Diagram
Short Description of the project

The Kaltech office network project involved designing and implementing a robust and secure network infrastructure for a small office with four distinct rooms: an IT Room, a Software Development Room, a Meeting Room, and a Finance Room. I configured and deployed Cisco routers (2911) and switches (2960) to ensure reliable, high-performance connectivity throughout the office. Wireless LAN Controllers (WLC-3504) and Access Points (Lightweight APs) were implemented to provide seamless Wi-Fi coverage across all rooms, supporting employee devices and guest access. I deployed a Cisco firewall (ASA 5505) to enhance network security to protect the network from external threats. Additionally, IP phones were integrated into the network for efficient employee communication. VLANs were created to segment the network traffic, improving security and optimizing performance for each department. The project showcased my ability to design, implement, and manage a comprehensive network solution tailored to meet the needs of a small office environment, ensuring scalability, security, and efficient communication.

Configuring the Network

Creating VLANs

Creating VLANs (Virtual Local Area Networks) is an effective way to segregate a network into smaller, isolated segments. By dividing a network into VLANs, organizations can enhance security, reduce broadcast traffic, and improve network performance. Each VLAN acts as a separate logical network, ensuring that devices within different VLANs cannot communicate directly unless specifically configured to do so. This segmentation helps contain potential security threats, limits the scope of network issues, and allows for more efficient traffic management. VLANs also make network administration easier by grouping devices based on function, department, or project rather than physical location.

The importance of creating VLANs to segregate a network lies in enhancing security, improving performance, and simplifying network management. By isolating network traffic within VLANs, sensitive data can be kept separate, reducing the risk of unauthorized access between different segments. VLANs also reduce broadcast traffic, preventing unnecessary congestion and optimizing bandwidth usage, which boosts overall network performance. Additionally, network segmentation allows for easier management and troubleshooting, as issues can be confined to specific VLANs. VLANs enable organizations to structure their networks based on functional or departmental needs, making it more scalable, flexible, and easier to manage.

To Create VLANs we configure the following settings on our Core Switch in the IT Dept.

en
conf t
 vlan 10
 name “Management VLAN”
 vlan 20
 name “Accounts Dept.”
 vlan 30 
 name “Software Dept.”
 vlan 40
 name “Meeting Room”
 vlan 50
 name “Misc. Vlan”

Each of these VLANs will be configured in each of our rooms. The VLAN 10 will be the Native Management VLAN where all traffic will be permitted. The VLAN 20 will be assigned to the Accounts Dept. The VLAN 30 will be assigned to the Software Dept. The VLAN 40 will be assigned to the Meeting Room. The VLAN 50 will be used to store all unused ports for security purposes.

Configuring VTP

VTP (VLAN Trunking Protocol) is a Cisco proprietary protocol used to manage and propagate VLAN (Virtual Local Area Network) information across a network of switches. It simplifies VLAN management by allowing a central switch (known as the VTP server) to create, modify, and delete VLANs, and automatically propagate these changes to other switches in the same VTP domain. The VTP domain used in this example is kaltech.com.

The importance of VTP (VLAN Trunking Protocol) in a network lies in its ability to simplify and streamline VLAN management across multiple switches. VTP allows network administrators to create, modify, and delete VLANs from a central switch (the VTP server), and automatically propagate these changes to all other switches in the same VTP domain (the VTP clients). This eliminates the need to manually configure VLANs on each switch, ensuring consistency and reducing the risk of configuration errors. VTP also helps in maintaining a unified VLAN structure across the network, making it easier to scale, manage, and troubleshoot large networks. By centralizing VLAN configuration, VTP improves network efficiency, reduces administrative overhead, and enhances the overall security and performance of the network.

to distribute our VLANs across all switches we will configure this network as follows

On Main Switch in Accounts Dept. , Software Dept. and Meeting Room

Shutting Down Inactive Ports

Shutting down inactive ports on network switches is a crucial practice for improving security, performance, and management. By disabling unused ports, organizations reduce the risk of unauthorized access, prevent network loops, and avoid unnecessary network traffic. This also helps in simplifying network troubleshooting, conserving bandwidth, and ensuring better energy efficiency. Additionally, keeping inactive ports disabled can assist in meeting compliance requirements and protecting the network from potential attacks. Overall, it enhances both the security posture and operational efficiency of the network.

Configuring Trunk Ports to all other Switches and Routers

Configuring all ports between switches and routers as trunk ports is crucial for efficiently managing multiple VLANs in a network. Trunk ports allow the transmission of traffic from several VLANs over a single connection, reducing the need for multiple physical links and simplifying network scalability. This setup is essential for enabling inter-VLAN routing on routers and ensures VLAN traffic is properly isolated while maximizing network resources. Additionally, it makes network expansion easier, as new VLANs can be added without reconfiguring each port, improving both network performance and management.

To configure trunk ports between the Core Switch in the IT Room to the Router in the IT Room and the Main Switches in the IT Room, Accounts, Software, and Meeting Room. We have to configure the following

The Network is Configured as

  1. Fast Ethernet 0/1 is connected to the core Router in the IT Room
  2. Fast Ethernet 0/2 is connected to the core Switch in the IT Room
  3. Fast Ethernet 0/3 is connected to the core Switch in the Accounts Room
  4. Fast Ethernet 0/4 is connected to the core Switch in the Software Room
  5. Fast Ethernet 0/5 is connected to the core Switch in the Meeting Room

en
conf t
interface FastEthernet0/1 — (Port Connected to Core Router)
switchport trunk native vlan 10
switchport trunk allowed vlan 10,20,30,40,60
switchport mode trunk 

interface Fa0/2 — (Port Connected to Core Switch in IT Dept.)
switchport trunk native vlan 10
switchport trunk allowed vlan 10,60
switchport mode trunk
int fa0/3— (Port Connected to Core Switch in Accounts Dept.)
switchport mode trunk
switchport trunk native vlan 10
switchport trunk allowed vlan 20,60
end
int fa0/4— (Port Connected to Core Switch in Software Dept.)
switchport mode trunk
switchport trunk native vlan 10
switchport trunk allowed vlan 30,60
end 
int fa0/5— (Port Connected to Core Switch in Meeting Room)
switchport mode trunk
switchport trunk native vlan 10
switchport trunk allowed vlan 40,60
end

We also need to configure ports that are connected to the Core Switch in the IT Room. All the Switches in the Accounts, Software, and Meeting room are connected to the Core Switch in the IT Room using their Port Fast Ethernet 0/1.

Changing the Native VLAN

Changing the native VLAN is important for improving network security and managing traffic more effectively. By default, the native VLAN is set to VLAN 1, which is commonly targeted by attackers. Changing the native VLAN to a different, unused VLAN helps reduce security risks, such as VLAN hopping attacks. Additionally, it allows for better traffic management by segregating untagged traffic and ensuring it is associated with a specific VLAN, improving overall network organization and performance. This practice also helps prevent potential network issues and conflicts related to the default VLAN.

We changed the Native VLAN in the Previous Heading. we used the command

switchport trunk native vlan 10

to change the native VLAN.

Configuring Access Ports to Edge Devices

Configuring access ports for edge devices is essential for ensuring proper network segmentation and security. Access ports are dedicated to a single VLAN, providing a clear boundary between different segments of the network. By assigning edge devices, such as computers or printers, to access ports, administrators can control traffic flow and ensure that devices are isolated within their respective VLANs. This configuration helps improve network performance, enhances security by limiting access to only authorized devices, and simplifies troubleshooting by clearly defining where devices are connected in the network.

Configuring Router On a Stick for InterVLAN Routing

A router on a Stick is an important method for inter-VLAN routing because it enables communication between multiple VLANs using a single physical interface on the router. By configuring subinterfaces on a single router interface, each assigned to a different VLAN, this setup allows the router to route traffic between VLANs without requiring multiple physical interfaces. This approach is cost-effective, as it reduces the need for additional router interfaces and simplifies network infrastructure. It also enhances scalability and flexibility, making it easier to manage and expand VLAN configurations as the network grows. Overall, Router on a Stick provides an efficient solution for inter-VLAN communication in smaller to medium-sized networks.

To Configure inter-VLAN routing we need to apply some configurations on the Core Router in the IT Dept. These Configurations are

en 
 conf t

 int g0/0 (Outside towards firewall)
 no shut

 int g0/1 (Inside Port)
ip address 192.168.1.1 255.255.255.0
 no shut

 int g0/1.10
 encapsulation dot1Q 10 native
ip address 192.168.1.2 255.255.255.0
 no shut

 int g0/1.20
 encapsulation dot1Q 20
ip address 192.168.2.1 255.255.255.0
 no shut

 int g0/1.30
 encapsulation dot1Q 30
ip address 192.168.3.1 255.255.255.0
 no shut

 int g0/1.40
 encapsulation dot1Q 40
ip address 192.168.4.1 255.255.255.0
 no shut

 int g0/1.50
 encapsulation dot1Q 50
 shut

int g0/1.60
encapsulation dot1Q 60
ip address 192.168.6.1 255.255.255.0
no shut

The g0/0 port is connected to a firewall which in turn is connected to the internet. The g0/1 port is connected to the internal network and all the sub-interfaces are responsible for routing traffic from their vlans.

  1. The int g0/1.10 will route traffic coming from the Core Switch’s VLAN 10 to other VLANs (20,30 and 40) and the Internet. 
  2. The int g0/1.20 will route traffic coming from the Core Switch’s VLAN 20 to other VLANs (10,30 and 40) and the Internet.
  3. The int g0/1.30 will route traffic coming from the Core Switch’s VLAN 30 to other VLANs (20,10 and 40) and the Internet.
  4. The int g0/1.40 will route traffic coming from the Core Switch’s VLAN 40 to other VLANs (20,30 and 10) and the Internet.
  5. The int g0/1.50 will be shut down to avoid traffic from Misc. VLAN.
  6. The int g0/1.60 will route traffic coming from the VOICE VLAN 60 to other VLANs (10,20,30 and 40) and the Internet.

Configuring DHCP

A DHCP (Dynamic Host Configuration Protocol) server is a network service that automatically assigns IP addresses and other related configuration settings to devices (like computers, printers, switches, etc.) on a network. This eliminates the need for manual IP address configuration, making network management more efficient and reducing the chance of address conflicts or errors.

In Our case, the Core Router will act as the DHCP Server.

The importance of DHCP (Dynamic Host Configuration Protocol) lies in its ability to automate the process of assigning IP addresses and network configurations to devices on a network. This eliminates the need for manual configuration, reducing errors and administrative workload. DHCP ensures efficient use of IP address space, prevents address conflicts, and simplifies network management, especially in large or dynamic environments. It is crucial for scalability, as it allows new devices to automatically receive an IP address and network settings, making it essential for modern networks that support numerous devices.

To Configure DHCP on Core Router, we run the following commands

After setting up all this configuration the Core Router will be the DHCP Server which will provide an IP address to the core switch in our network.

The Core Switch will act as a DHCP Relay and transfer all the DHCP Traffic to the Core Router. To configure Core Switch as DHCP Relay we apply:

They enable voice communication over the Internet or private networks, reducing the need for traditional telephone lines and lowering operational costs. By leveraging existing data networks, IP phones offer scalability, flexibility, and advanced features such as call forwarding, voicemail, and conferencing. Their ability to easily integrate with other communication systems and provide high-quality audio makes them indispensable for businesses aiming to streamline operations and improve communication efficiency. Additionally, IP phones support remote work environments, allowing employees to connect from virtually anywhere with an internet connection.
